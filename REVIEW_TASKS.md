# Предложенные задачи после быстрого ревью

## 1) Опечатка/нейминг
**Задача:** унифицировать название обработчика уведомлений об изменении сроков: в `assignments/views.py` вызывается `process_deadline_change`, а в `telegram/notifications.py` объявлен `process_deadline_extensions`.

**Почему важно:** сейчас это выглядит как опечатка в API между модулями и ведёт к `ImportError` при попытке отправить уведомление из UI.

## 2) Исправление ошибки
**Задача:** добавить импорт `get_object_or_404` в `assignments/views.py`.

**Почему важно:** функция `assignment_detail` использует `get_object_or_404`, но он не импортирован, что приводит к `NameError` при открытии карточки поручения.

## 3) Исправление комментария/несоответствия документации
**Задача:** уточнить комментарий в `assignments/views.py` у расчёта `days_left`.

**Текущее состояние:** комментарий `# отрицательное если просрочено` грамматически и терминологически неточен (речь о *значении* числа дней).

**Предложение:** заменить на более точный вариант, например: `# значение < 0, если срок уже просрочен`.

## 4) Улучшение теста
**Задача:** добавить интеграционный тест для POST-ветки в `assignment_bulk_action` c действием `notify_deadline`.

**Почему важно:** тест должен проверять корректный вызов уведомления и отсутствие падения из-за ошибок импорта/нейминга обработчика. Сейчас файлы тестов в приложениях содержат только шаблонные заглушки `# Create your tests here.`.
